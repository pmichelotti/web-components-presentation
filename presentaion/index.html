Introduction

The Early Days of HTML

During the days of AOL dialup and animated gifs as ubiquitous as cat videos HTML was in its infancy as a markup language and browsers were experimenting with the capabilities which could be added.

The Blink Tag!

One of these experiments was the failed blink tag implemented in Netscape Navigator. This tag caused the contained text to blink, as the name would suggest, at a regular interval. It delighted ever 14 year old aspiring web developer who discovered it and appeared right alongside the visit count on nearly every site.

Markup Standardization

Some time later browsers moved away from some of these custom tags and HTML was generalized as a markup language optimized for document layout. This gave us markup such as ‘div’, ‘span’, ‘p’, ‘table’, etc, which could all be used in conjunction with style sheets to produce the web version of a brochure.

The Current Situation

Now pages behaving as applications is becoming increasingly common, and this can produce tension between the needs of the application and the document-centric nature historically present in HTML markup

Back to the Blink Tag

					
I might could blink
While the blink tag is laughable today, it’s actually particularly useful to have such a capable tag if it serves your purpose. Let’s take a look at an implementation of the functionality this tag provided which you might produce when faced with similar requirements.

Some Problems With This Implementation

  1. First, the markup is not particularly meaningful. It is a span - which to start with is a meaningless piece of markup
  2. The only semantically meaningful item here is the class attribute
  3. The javascript looks for spans with a "blink" class. This means that you need to ensure that no other spans on your site ever have a "blink" class unless you want them to blink
  4. You also need to ensure that no other Javascript or CSS which you write or include acts upon things with a "blink" class

Enter Web Components

The blink tag did not have these issues. You could have spans, or anything else, with a "blink" class, which would have no effect on the blink tag or its behavior. As an application developer, what you want to be able to do is reproduce the blink tag and cause the tag to encapsulate the functionality which Netscape imbued the blink tag with - and with Web Components, you can!
